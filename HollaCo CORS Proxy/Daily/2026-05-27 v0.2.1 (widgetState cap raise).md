# v0.2.1 — widgetState cap raise (20K → 64K)

Single-constant change to unblock HollaCo Command Center widget Phase 7.1 (wiki grounding). The widget now feeds Claude up to ~40K chars of wiki page bodies on top of the existing ~20K of widget-state context, so the proxy's POST `/api/claude` route needs to accept payloads up to ~60K.

#project/cors-proxy #release/v0.2.1 #topic/ai

## What changed

| File | Change |
|---|---|
| `src/functions/claude.js:6` | `MAX_WIDGET_STATE_CHARS = 20000` → `64000` |
| `src/__tests__/claude.test.js` | Updated 1 existing test, added 1 new test (accept at exactly 64000) |
| `package.json` | Version bump |

That's the entire diff.

## Why 64000

Phase 7.1 spec budget:
- ~20K base widget-state markdown
- ~40K wiki content (5 pages × 8K chars each, lazy-fetched only when prompt has lexical matches)
- ~4K headroom for section separators + future growth

64K covers it with breathing room. Could raise further later if Phase 7.2/7.3 wants more context; Anthropic's input window is 200K so we have a long runway before this becomes the limiting factor. Token cost remains the actual constraint (~$0.18/query at 60K input + Sonnet 4.6 pricing).

## Deploy

Pushed to main → GitHub Actions OIDC deploy → smoke verified with a 50K-char `widgetState` POST returning 200 instead of the v0.2.0 `widgetState too long` rejection.
